## Components — Convenciones

- Organizados por **dominio/feature**: `campaigns/`, `character/`, `combat/`, `creation/`, `dice/`, `inventory/`, `notes/`, `settings/`, `compendium/`.
- `ui/` contiene componentes reutilizables del design system con barrel export en `index.ts`. Nuevos componentes UI deben exportarse ahí.
- **Patrón**: pantallas (en `app/`) orquestan → componentes renderizan → stores/hooks proveen estado.
- **Estilos**: preferir NativeWind `className`. Solo usar `StyleSheet.create` para estilos dinámicos o valores calculados en runtime.
- **Tema**: usar `useTheme()` para tokens de color. Paleta D&D definida en `tailwind.config.js`.
- Archivos de componentes en **PascalCase**: `CharacterSheet.tsx`, `DiceRoller.tsx`.
- Imports con path aliases: `@components/`, `@hooks/`, `@stores/`, `@types/`, etc.
