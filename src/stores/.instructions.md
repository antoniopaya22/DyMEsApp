## Stores — Convenciones

- Cada store es un dominio aislado creado con `create` de Zustand.
- **Persistencia manual**: usar helpers de `@utils/storage` (setItem, getItem, removeItem). NUNCA usar Zustand persist middleware.
- **Character store** es especial: compuesto por slices en `src/stores/characterStore/`. Para añadir funcionalidad, crear o editar un slice — no inflar `index.ts`.
- Helpers puros en archivos `*Helpers.ts` separados. Los stores solo orquestan llamadas, no contienen lógica de negocio compleja.
- Acceso a DB siempre via `@/services/supabaseService` — nunca importar el supabase client directamente.
- Imports con path aliases: `@stores/`, `@utils/`, `@types/`, etc.
